Section I: Rasterization

Part 1: Rasterizing single-color triangles

  1. Walk through how you rasterize triangles in your own words.

The high-level overview of how we rasterized the triangle is (1) sort the vertices in counter clockwise order, (2) loop through each point the bounding box of the triangle, (3) check if it’s inside the triangle, and (4) sample if it’s inside the triangle.

We sorted the vertices of the triangle in a counterclockwise order by finding the centroid (by averaging the coordinate values), finding the angle theta between each vertex and the centroid, and then sorting by theta. This allows us to sort the vertices and arrange them in counterclockwise order. We then calculated the normal vector for each edge. Then, we loop through each point in the bounding box of the triangle. If the dot product of the point’s vector and all of the normal vectors are less than 0, we know that the point is in the triangle and it is sampled.

  1. Explain how your algorithm is no worse than one that checks each sample within the bounding box of the triangle.

What we did was check if the point is inside the triangle and then take a sample, instead of sampling and then checking if the point is inside the triangle. This allows us to eliminate unnecessary samping, thus making our algorithm no worse than one that checks each sample within the bounding box of the triangle.

Part 2: Antialiasing triangles

  1. Walk through your supersampling algorithm and data structures. Why is supersampling useful? What modifications did you make to the rasterization pipeline in the process? Explain how you used supersampling to antialias your triangles.

The two main super sampling data structures we used are sample_buffer and rgb_framebuffer_target. While the rgb_framebuffer_target size is set to 3 * width * height, sample_buffer size is set to sample_rate * width * height. That way, when we change the sampling rate for super sampling, we have sample_rate samples per pixel. When we take the super samples, we save the values in sample_buffer then map those values to rgb_framebuffer_target in resolve_to_framebuffer. The algorithm we used for supersampling was more or less what we learned in lecture — taking more samples per pixel then taking the sample averages when we’re getting the values for the frame buffer.

Supersampling is useful for reducing jaggies and aliasing as seen along the edges of the triangles in task 1. Using supersampling enables us to sample multiple locations within the pixel (depending on the sampling rate) and average the values depending on whether the sample falls within the triangle. Since pixels with fewer samples inside the triangle will have a lower saturation, this "smooths out" the edges and helps with anti-aliasing, however the trade off is that this becomes very costly.

Part 4: Barycentric coordinates

  1. Explain barycentric coordinates in your own words and use an image to aid you in your explanation. One idea is to use a svg file that plots a single triangle with one red, one green, and one blue vertex, which should produce a smoothly blended color triangle.


        Barycentric coordinates are useful when we have 3 given coordinates like vertices, and we want to obtain smooth varying values across a surface between these given coordinates through interpolation.

For example, the three points A (x0, y0), B (x1, y1), C (x2, y2) representing the vertices of a triangle, where A is red, B is green and C is blue, barycentric coordinates of an arbitrary point V = (x,y) in the triangle, will linearly interpolate the value (color) at vertices to find the value (color) of the point. The color of V = alpha * A + beta *B + gamma * C.

We can find alpha, beta and gamma by:

  • alpha = Lbc(x,y)/Lbc(x_A, y_A)
  • beta = Lca(x,y)/Lca(x_B,y_B)
  • gamma = 1 - alpha - beta

This can be seen from the result below where the triangle now has a smoothly blended mix of the three colors.

Part 5: "Pixel sampling" for texture mapping

  1. Explain pixel sampling in your own words and describe how you implemented it to perform texture mapping. Briefly discuss the two different pixel sampling methods, nearest and bilinear.

Pixel sampling involves determining the color values for pixels in the final image by selecting a subset of pixels from an original image.

We implemented two methods, nearest and bilinear. For nearest sampling, we updated the sample_nearest method in texture.cpp. It takes the texture coordinates (u, v) for a pixel and rounds it to the nearest texel coordinates in the texture map and finally it selects the color of this nearest texel for the pixel.

For bilinear sampling, we updated the sample_bilinear method in texture.cpp. We find the four nearest texels to the texture coordinates (u, v) of a given pixel and interpolate the color value of the target pixel. This method will result in a smoother transition between texels compared to the nearest method.

Part 6: "Level sampling" with mipmaps for texture mapping

  1. Explain level sampling in your own words and describe how you implemented it for texture mapping.

In level sampling, we sample from the levels of a mipmap that best matches the pixel coverage of a texture on the screen. Mipmaps are a series of progressively lower resolution images of the texture image. This is done by taking  a texture image file, low-pass filtering it and downsampling it recursively. Level sampling is useful as it significantly helps with antialiasing and improves rendering performance.

Firstly, we start by computing mipmap level D in get_level of texture.cpp. This is done by calculating difference vectors for the texture coordinates. These vectors represent the change in texture coordinates between adjacent pixels on the screen. We then scale the difference vectors by width and height, before finally calculating L by the maximum of the norm of the scaled difference vectors and D is calculated by taking the log base 2 of L. The SampleParams object is finally put into the texture sampling function to find the color at the pixel.  

  1. You can now adjust your sampling technique by selecting pixel sampling, level sampling, or the number of samples per pixel. Describe the tradeoffs between speed, memory usage, and antialiasing power between the three various techniques.

In terms of speed, nearest pixel sampling and level sampling (with nearest pixel sampling) are both faster than bilinear sampling (supersampling) since bilinear sampling requires us to calculate the values of the neighbors and this adds time to the process. Nearest pixel sampling is slightly faster than level sampling as level sampling requires additional time to calculate the mipmap level and sample across levels.

The method of number of samples per pixel (Supersampling) takes up the most memory as it samples more points and goes through these points multiple times (sample_rate).

In terms of antialiasing power, nearest pixel sampling has the worst performance since it just takes the value of the closest neighbor but many texels can be in the same pixel footprint, such as when the image is highly minified. This leads to much more obvious aliasing. Bilinear sampling performs much better for antialiasing. Level sampling also helps with antialiasing, but it performs worse than Bilinear sampling.  As such, the slowest method is bilinear level interpolation with bilinear pixel interpolation, since there are more calculations of the neighboring pixels and across the levels.
